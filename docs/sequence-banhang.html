Sơ đồ tuần tự - Chức năng Bán hàng

Kiến trúc Client/Server qua socket DTO. Render bằng PlantUML qua kroki.io.
Ký hiệu: actor = người dùng  |  boundary = giao diện Swing  |  control = Service / facade  |  participant = truyền tải socket  |  entity = DAO Layer  |  database = MariaDB.
Đang tải sơ đồ từ kroki.io...
@startuml
title Sơ đồ tuần tự - Chức năng Bán hàng (Client/Server)

skinparam shadowing false
skinparam roundCorner 8
skinparam ParticipantPadding 14
skinparam BoxPadding 10
skinparam sequenceMessageAlign center
skinparam DefaultFontSize 14
skinparam TitleFontSize 18
skinparam SequenceGroupFontSize 16
skinparam SequenceMessageFontSize 14
skinparam ParticipantFontSize 14
skinparam ActorFontSize 14
skinparam DatabaseFontSize 14
skinparam ArrowThickness 1.2

actor "Nhân viên" as NV
box "Client" #EAF4FF
boundary "BanHang_GUI" as GUI
control "ClientService" as Svc
participant "ClientSocket" as Sock
end box
box "Server" #FFF6E5
participant "ClientHandler" as Handler
control "SanPhamService" as SPSvc
control "LoSanPhamService" as LoSvc
control "QuyCachDongGoiService" as QCSvc
control "KhuyenMaiService" as KMSvc
control "KhachHangService" as KHSvc
control "BangGiaService" as BGSvc
control "HoaDonService" as HDSvc
entity "DAO Layer\n(JPA / Hibernate)" as DAO
database "MariaDB" as DB
end box

== 1. Nhân viên nhập mã SP / Số ĐK ==
NV -> GUI : 1: Nhập mã SP / Số ĐK
GUI -> Svc : 1.1: getProductByCode(maSP)
Svc -> Sock : 1.1.1: Request(SANPHAM_LAY_THEO_MA, maSP)
Sock -> Handler : 1.1.2: send(Request)
Handler -> SPSvc : 1.1.3: laySanPhamTheoMa(maSP)
SPSvc -> DAO : 1.1.4: timSanPhamTheoMa(maSP)
DAO -> DB : 1.1.5: SELECT FROM SanPham WHERE maSanPham = ?
DB --> DAO : 1.1.6: row
DAO --> SPSvc : 1.1.7: SanPham
SPSvc --> Handler : 1.1.8: SanPhamDTO
Handler --> Sock : 1.1.9: Response(SanPhamDTO)
Sock --> Svc : 1.1.10: Response
Svc --> GUI : 1.2: SanPhamDTO

alt Tìm thấy sản phẩm
  GUI -> Svc : 1.3: getLotsByProduct(maSP)
  Svc -> Sock : 1.3.1: Request(LO_LAY_THEO_SAN_PHAM, maSP)
  Sock -> Handler : 1.3.2: send(Request)
  Handler -> LoSvc : 1.3.3: layLoConHangTheoSanPham(maSP)
  LoSvc -> DAO : 1.3.4: timLoConHang(maSP)
  DAO -> DB : 1.3.5: SELECT lô còn HSD AND soLuongTon > 0
  DB --> DAO : 1.3.6: rows
  DAO --> LoSvc : 1.3.7: List
  LoSvc --> Handler : 1.3.8: List
  Handler --> Sock : 1.3.9: Response(List)
  Sock --> Svc : 1.3.10: Response
  Svc --> GUI : 1.4: dsLoSanPham

  GUI -> Svc : 1.5: getPackagingRulesByProduct(maSP)
  Svc -> Sock : 1.5.1: Request(QUYCACH_LAY_THEO_SAN_PHAM, maSP)
  Sock -> Handler : 1.5.2: send(Request)
  Handler -> QCSvc : 1.5.3: layQuyCachTheoSanPham(maSP)
  QCSvc -> DAO : 1.5.4: SELECT FROM QuyCachDongGoi WHERE maSanPham = ?
  DAO -> DB : 1.5.5: SELECT
  DB --> DAO : 1.5.6: rows
  DAO --> QCSvc : 1.5.7: List
  QCSvc --> Handler : 1.5.8: List
  Handler --> Sock : 1.5.9: Response(List)
  Sock --> Svc : 1.5.10: Response
  Svc --> GUI : 1.6: dsQuyCach

  GUI -> Svc : 1.7: getActivePromotionDetailsByProduct(maSP)
  Svc -> Sock : 1.7.1: Request(KHUYENMAI_CHI_TIET_DANG_AP_DUNG, maSP)
  Sock -> Handler : 1.7.2: send(Request)
  Handler -> KMSvc : 1.7.3: timKMSanPham(maSP)
  KMSvc -> DAO : 1.7.4: timKMTheoSanPham(maSP)
  DAO -> DB : 1.7.5: SELECT FROM ChiTietKhuyenMaiSanPham JOIN KhuyenMai
  DB --> DAO : 1.7.6: rows
  DAO --> KMSvc : 1.7.7: List
  KMSvc --> Handler : 1.7.8: List
  Handler --> Sock : 1.7.9: Response(List)
  Sock --> Svc : 1.7.10: Response
  Svc --> GUI : 1.8: kmSanPham

  GUI -> Svc : 1.9: getBangGiaDangHoatDong()
  Svc -> Sock : 1.9.1: Request(BANGGIA_LAY_HOAT_DONG)
  Sock -> Handler : 1.9.2: send(Request)
  Handler -> BGSvc : 1.9.3: layBangGiaHoatDong()
  BGSvc -> DAO : 1.9.4: truy vấn BangGia đang hoạt động
  DAO -> DB : 1.9.5: SELECT FROM BangGia WHERE hoatDong = true
  DB --> DAO : 1.9.6: row
  DAO --> BGSvc : 1.9.7: BangGia
  BGSvc --> Handler : 1.9.8: BangGiaDTO
  Handler --> Sock : 1.9.9: Response(BangGiaDTO)
  Sock --> Svc : 1.9.10: Response
  Svc --> GUI : 1.10: bangGia

  GUI --> NV : 1.11: Hiển thị thông tin sản phẩm
else Không tìm thấy / hết hàng / thiếu quy cách
  GUI --> NV : 1.12: Thông báo lỗi
end

== 2. Nhân viên nhập số lượng và kiểm tra khuyến mãi ==
NV -> GUI : 2: Nhập số lượng
GUI -> GUI : 2.1: Kiểm tra số lượng từ dsLo
alt Đủ số lượng
  GUI -> GUI : 2.2: Cập nhật tổng tiền
  GUI -> Svc : 2.3: getActiveKhuyenMai(tongTien)
  Svc -> Sock : 2.3.1: Request(KHUYENMAI_LAY_DANG_AP_DUNG, tongTien)
  Sock -> Handler : 2.3.2: send(Request)
  Handler -> KMSvc : 2.3.3: timKhuyenMaiHoaDonPhuHop(tongTien)
  KMSvc -> DAO : 2.3.4: timKMHoaDon(tongTien)
  DAO -> DB : 2.3.5: SELECT KM hóa đơn còn hiệu lực
  DB --> DAO : 2.3.6: rows
  DAO --> KMSvc : 2.3.7: List
  KMSvc --> Handler : 2.3.8: List
  Handler --> Sock : 2.3.9: Response(List)
  Sock --> Svc : 2.3.10: Response
  Svc --> GUI : 2.4: KhuyenMaiHoaDon
  GUI --> NV : 2.5: Hiển thị thành tiền
else Không đủ số lượng
  GUI --> NV : 2.6: Thông báo không đủ tồn
end

== 3. Nhân viên nhập SĐT khách hàng ==
NV -> GUI : 3: Nhập SĐT khách
GUI -> Svc : 3.1: getKhachHangByPhone(sdt)
Svc -> Sock : 3.1.1: Request(KHACHHANG_TIM_THEO_SDT, sdt)
Sock -> Handler : 3.1.2: send(Request)
Handler -> KHSvc : 3.1.3: timKhachHangTheoSDT(sdt)
KHSvc -> DAO : 3.1.4: timTheoSDT(sdt)
DAO -> DB : 3.1.5: SELECT FROM KhachHang WHERE soDienThoai = ?
DB --> DAO : 3.1.6: row | null
DAO --> KHSvc : 3.1.7: KhachHang
KHSvc --> Handler : 3.1.8: KhachHangDTO
Handler --> Sock : 3.1.9: Response(KhachHangDTO)
Sock --> Svc : 3.1.10: Response
Svc --> GUI : 3.2: KhachHang
alt Có khách hàng
  GUI --> NV : 3.3: Hiển thị thông tin khách hàng
else Không có
  GUI -> GUI : 3.4: Set khách vãng lai
end

== 4. Nhân viên nhập tiền khách đưa ==
NV -> GUI : 4: Nhập tiền khách đưa
GUI -> GUI : 4.1: Kiểm tra tiền hợp lệ
alt Tiền hợp lệ
  GUI --> NV : 4.2: Tính và hiển thị tiền thừa
else Tiền không hợp lệ
  GUI --> NV : 4.3: Yêu cầu nhập lại
end

== 5. Nhân viên bấm thanh toán ==
NV -> GUI : 5: Bấm Thanh toán
GUI -> Svc : 5.1: taoMaHoaDon()
Svc -> Sock : 5.1.1: Request(HOADON_TAO_MA)
Sock -> Handler : 5.1.2: send(Request)
Handler -> HDSvc : 5.1.3: taoMaHoaDonTuDong()
HDSvc -> DAO : 5.1.4: taoMaHoaDon()
DAO -> DB : 5.1.5: SELECT MAX(maHoaDon)
DB --> DAO : 5.1.6: maxId
DAO --> HDSvc : 5.1.7: maHD
HDSvc --> Handler : 5.1.8: maHD
Handler --> Sock : 5.1.9: Response(maHD)
Sock --> Svc : 5.1.10: Response
Svc --> GUI : 5.2: maHD

GUI -> GUI : 5.3: Build HoaDon + ChiTietHoaDon
GUI -> Svc : 5.4: createHoaDon(hd)
Svc -> Sock : 5.4.1: Request(HOADON_THEM, HoaDonCreateUpdateDTO)
Sock -> Handler : 5.4.2: send(Request)
Handler -> HDSvc : 5.4.3: themHoaDon(dto)
HDSvc -> DAO : 5.4.4: themHoaDon(entity)
DAO -> DB : 5.4.5: BEGIN TX
DAO -> DB : 5.4.6: INSERT HoaDon
loop Mỗi chi tiết hóa đơn
  DAO -> DB : 5.4.7: INSERT ChiTietHoaDon
  DAO -> DB : 5.4.8: UPDATE LoSanPham SET soLuongTon = soLuongTon - ?
end
DAO -> DB : 5.4.9: COMMIT
DB --> DAO : 5.4.10: OK
DAO --> HDSvc : 5.4.11: true
HDSvc --> Handler : 5.4.12: true
Handler --> Sock : 5.4.13: Response(success)
Sock --> Svc : 5.4.14: Response
Svc --> GUI : 5.5: ok

opt Có khuyến mãi hóa đơn
  GUI -> Svc : 5.6: reduceKhuyenMaiQuantity(maKM)
  Svc -> Sock : 5.6.1: Request(KHUYENMAI_GIAM_SO_LUONG, maKM)
  Sock -> Handler : 5.6.2: send(Request)
  Handler -> KMSvc : 5.6.3: giamSoLuongKhuyenMai(maKM)
  KMSvc -> DAO : 5.6.4: capNhatSoLuongKM(maKM)
  DAO -> DB : 5.6.5: UPDATE KhuyenMai SET soLuongKhuyenMai = soLuongKhuyenMai - 1
  DB --> DAO : 5.6.6: OK
  DAO --> KMSvc : 5.6.7: true
  KMSvc --> Handler : 5.6.8: true
  Handler --> Sock : 5.6.9: Response(success)
end

alt Lưu thành công
  GUI --> NV : 5.7: Thông báo bán hàng thành công
else Lỗi
  GUI --> NV : 5.8: Thông báo bán hàng có lỗi
end

@enduml
